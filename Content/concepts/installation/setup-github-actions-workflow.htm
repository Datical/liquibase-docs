GitHub Actions Workflow Setup

If you don’t want to install locally on your laptop or use a remote host that will run the software for you, you can try the GitHub Actions workflow.

GitHub Actions is a way to automate, customize, and execute your software development workflows in a repository. You can create and share actions to perform any job, including CI/CD, and combine actions in a customized workflow.

  • A workflow is an automated procedure that you add to your repository. Workflows are made up of one or more jobs and can be scheduled or triggered by an event. The workflow can be used to build, test, package, release, or deploy a project on GitHub.
  • A job is a set of steps that is executed on the same runner.

For more information about GitHub Actions, see Introduction to GitHub Actions.

Prerequisites

Ensure that you have a GitHub account.

For more information on how to create a GitHub account, see Signing Up for GitHub.

How to use the GitHub Actions repository

To use the GitHub Actions repository, follow these steps:

  1. Open and fork the repository by selecting the Fork button in the upper-right corner of the page. For more information about forking, see Forking Projects.
  2. In the browser, go to your GitHub githubActions-liquibase repository, and then under the Workflow section, go to Actions.
  3. Choose the needed workflow, and then select run workflow.

To adjust your workflow, go to the workflow script in githubActions-liquibase/.github/workflows/<workflow name>.yaml and make the needed changes. You can add more commands and flags to the workflow script.

To add and change a or edit the file, modify them in the repository and then commit your changes without installing the files locally.
